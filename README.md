## ğŸ”¬ Methodology / å®éªŒæ–¹æ³•è®º

[English]
To validate the effectiveness of the proposed architecture without compromising network security, we developed a **Trace-driven Simulator** based on real-world data:
1.  **Data Source**: Historical Carbon Intensity data retrieved from **NESO (UK)** and **Ember** (Global Data) was used to model the energy environment.
2.  **Counterfactual Framework**: For every DNS request, the simulator calculates two outcomes simultaneously:
    *   **Baseline**: Routes to the physically nearest node (Latency minimization).
    *   **Experiment**: Routes to the greenest node within an acceptable latency tolerance (Carbon minimization).
3.  **Verifiability**: The `Audit_Proof` is generated using a hash chain mechanism to ensure the scheduling path is tamper-proof.
